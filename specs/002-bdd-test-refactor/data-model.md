# Data Model: BDD Test Refactor

**Date**: 2026-03-30
**Feature**: 002-bdd-test-refactor

## Entities

### BDD Case File

A markdown file containing structured BDD scenarios for one functional area.

**Attributes**:
- `功能` (Feature): Module name + feature description (Chinese)
- `模块` (Module): Functional module identifier (e.g., 网关管理, 资源配置)
- `页面` (Page): URL path pattern (e.g., `/:gatewayId/resource`)
- `优先级` (Priority): P1, P2, or P3
- `前置条件` (Prerequisites): Comma-separated list of required conditions
- `场景[]` (Scenarios): One or more BDD scenarios with 假设/当/那么 steps

**Identity**: File path (e.g., `test/bdd-cases/01-网关管理/01-创建普通网关.md`)

**Lifecycle**: Created once during Excel conversion → manually maintained → triggers script regeneration on change

**Relationships**:
- 1 BDD Case File → 1 Test Script (`.spec.js`)
- N BDD Case Files → 1 Module directory
- Each BDD Case File derives from multiple Excel Test Cases (N:1 consolidation)

---

### Test Script

A Playwright `.spec.js` file generated from a BDD case by the agent SKILL.

**Attributes**:
- File path mirroring BDD case path (e.g., `test/bdd-scripts/01-网关管理/01-创建普通网关.spec.js`)
- `test.describe` block matching `# 功能`
- `test()` blocks matching `## 场景`
- Environment variable references: `TEST_URL`, `TEST_USER`, `TEST_PASSWORD`, `TEST_COOKIE`, `TEST_GATEWAY_ID`

**Identity**: File path

**Lifecycle**: Generated by agent SKILL → committed to repo → regenerated when BDD case changes → executed by `make test-bdd`

**Relationships**:
- 1 Test Script ← 1 BDD Case File (source)
- All Test Scripts share `setup.js` / `teardown.js` / `helpers.js`

---

### Test Gateway

An ephemeral gateway created during test setup for mutating operations.

**Attributes**:
- `name`: Auto-generated (e.g., `testagent-<timestamp>`)
- `id`: Assigned by the platform after creation
- `type`: 普通网关 (standard)
- `visibility`: 不公开 (private)

**Identity**: Gateway ID (numeric, from URL)

**Lifecycle**: Created by `setup.js` → used by all mutating test scripts → deleted by `teardown.js`

**Relationships**:
- 1 Test Gateway per test suite run
- Referenced by all mutating test scripts via `TEST_GATEWAY_ID` env var

---

### Test Report

Output from Playwright Test runner after a complete suite execution.

**Attributes**:
- `results.json`: Machine-readable JSON with per-test pass/fail/error status
- `html/index.html`: Human-readable HTML report with screenshots
- Summary: total, passed, failed, error counts
- Exit code: 0 (all pass) or non-zero (any failure)

**Identity**: Timestamp-based output directory

**Lifecycle**: Generated after each `make test-bdd` run → not committed to repo (gitignored)

---

### SKILL Definition

Configuration file guiding the AI agent through BDD-to-script conversion.

**Attributes**:
- Location: `.agents/skills/bdd-test-gen/SKILL.md`
- Input parameters: `--url`, `--user`/`--password` or `--cookie`, `--case` or `--all`
- Workflow: Read BDD → Login → Navigate → Explore → Generate script → Verify → Save

**Identity**: File path

**Lifecycle**: Created once during refactor → maintained as the process evolves

## State Transitions

### BDD Case Lifecycle

```
[Excel Row] ---(conversion)--→ [BDD Case Draft] ---(review/edit)--→ [BDD Case Committed]
                                                                            |
                                                                    (case modified)
                                                                            ↓
                                                                    [Script Regeneration Needed]
                                                                            |
                                                                    (agent SKILL run)
                                                                            ↓
                                                                    [Script Updated & Committed]
```

### Test Run Lifecycle

```
[make test-bdd invoked with URL+credentials]
    → [setup.js: create test gateway]
    → [run scripts sequentially]
    → [teardown.js: delete test gateway]
    → [report generated]
    → [exit code returned]
```
