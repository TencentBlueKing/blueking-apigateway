# Contract: Test Report Format

**Version**: 1.0
**Date**: 2026-03-24

## Purpose

Defines the standard format for test run reports generated by the agent test runner.

## Report File Location

```
test/agent-tests/reports/YYYY-MM-DDTHH-MM-SS/
├── report.md           # The report file
└── screenshots/        # Screenshots captured during the run
    ├── case0-login-step1.png
    ├── case0-login-verify.png
    ├── case1-gateway-list-step1.png
    └── ...
```

## Report Format

```markdown
# Test Run Report

**Date**: [YYYY-MM-DD HH:MM:SS]
**Target**: [base URL]
**Auth Method**: [credentials | cookie]
**Duration**: [total time, e.g., "12m 34s"]

## Summary

| Metric   | Count |
|----------|-------|
| Total    | [N]   |
| Passed   | [N]   |
| Failed   | [N]   |
| Skipped  | [N]   |
| Errors   | [N]   |

## Results

### ✅ case0-login — PASSED (15s)

Steps completed: 3/3
Screenshots: [step1](screenshots/case0-login-step1.png), [verify](screenshots/case0-login-verify.png)

---

### ❌ case1-gateway-list-filters — FAILED (32s)

Steps completed: 2/4
**Failure at step 3**: Expected type filter dropdown to show "可编程网关" but dropdown was not found on the page.
Screenshots: [step1](screenshots/case1-step1.png), [step2](screenshots/case1-step2.png), [failure](screenshots/case1-failure.png)

---

### ⏭️ case2-create-gateway — SKIPPED

**Reason**: Prerequisite not met: "Gateway 'test-gw' must not exist"

---
```

## Status Icons

| Status  | Icon | Meaning |
|---------|------|---------|
| Passed  | ✅   | All steps executed and all verifications passed |
| Failed  | ❌   | A verification check did not match expected outcome |
| Skipped | ⏭️   | Prerequisites not met, case not executed |
| Error   | ⚠️   | Unexpected error during execution (browser crash, timeout, etc.) |
