# 功能: 在线调试 - API在线调试

**模块**: 在线调试
**页面**: /gateways/:id/online-debug
**优先级**: P1
**前置条件**: 已登录，且存在已配置的资源

## 场景: 发送调试请求

- **假设** 用户已登录并进入在线调试页面
- **当** 用户选择环境，选择请求资源，选择GET请求方法
- **并且** 选填Headers、Query、Body后点击"发送请求"
- **那么** 响应区域应显示Status为200
- **并且** Body中应返回正常的测试数据
- **并且** 使用POST请求方法同样可以成功发送请求并返回正确响应

## 场景: 设置请求参数

- **假设** 用户已进入在线调试页面
- **当** 用户选择"自定义应用"并填写应用信息
- **并且** 选择"自定义用户认证"并填写正确的认证信息
- **那么** 发送请求后应成功返回Status为200的响应
- **并且** 支持在资源列表下方输入关键字搜索资源
- **并且** 支持通过关键字搜索请求方法

## 场景: 查看调试响应

- **假设** 用户已发送调试请求
- **当** 后端服务返回异常（如500状态码）
- **那么** 响应区域应显示对应的错误Status状态码
- **并且** Body中应显示错误返回信息
